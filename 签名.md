## 接口签名规则
### 1. 前置准备
本地生成一组ECDSA公私钥，请求方使用私钥完成签名，服务端通过请求头传入的公钥进行验签。

### 2. header头 X-Params 组装规则
1. message 格式是 
```text
${钱包地址}${面额}${面额id}
```

2. message 需要进行 sha256.sum256 成 32位
3. 拿web3钱包对应的私钥对 message 签名，签名最后要 base64 encode
4. 组装 json 数据: `{"wallet_address":"${钱包地址}","money":"${面额}","money_id":"${面额id}","signature":"${上面生成的签名}"}`, 
最后对这个json字符串 base64 encode, 填装到 http 请求的 header X-Params 参数里. 将业务钱包相关信息组装为JSON结构，再对JSON字符串做Base64编码，赋值给请求头`X-Params`
5. 签名补充说明：
- BTC、LTC：使用 `github.com/btcsuite/btcd/btcec/v2/ecdsa.SignCompact` 签名，如果钱包地址是 taproot 类型的，请使用 `github.com/btcsuite/btcd/btcec/v2/schnorr.Sign` 签名
- ETH：使用 `github.com/ethereum/go-ethereum/crypto.Sign` 签名
- 如果 BTC 钱包是 taproot 类型的私钥签名前需要用包 `github.com/btcsuite/btcd/txscript.TweakTaprootPrivKey` 把私钥调整，然后用调整后的私钥进行签名

### 2. header头 X-Signature 签名生成逻辑
```go
var r *http.Request
var parts []string
params := r.Header.Get("X-Params")
nonce := r.Header.Get("X-Nonce")
parts = append(parts, params, nonce)
message := strings.Join(parts, "")
hash := sha256.Sum256([]byte(message))
signature, err := ecdsa.SignASN1(rand.Reader, ecdsaPrivateKey, hash[:])
signStr := hex.EncodeToString(signature)
```

### 3. 请求Header说明
| 请求头 | 说明 |
| ---- | ---- |
| Content-Type | 固定为 `application/json` |
| X-Nonce | 随机字符串，参与ECDSA接口签名报文拼接 |
| X-Signature | ECDSA私钥生成的十六进制签名字符串 |
| X-Public-Key | 本地ECDSA公钥，用于服务端验签 |
| X-Params | 钱包业务JSON整体Base64编码字符串 |
